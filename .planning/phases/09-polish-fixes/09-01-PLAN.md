**useAccessControl.ts (issue #6):**
1. In the `queryFn` for `freeReadsUsed` (lines 14-19), the current query fetches all `user_progress` rows and counts them with `data?.length`. A user reading the same book multiple times creates duplicate entries (or updates via upsert create one entry per book, but the query still counts all rows including potential duplicates from other sources).
2. Fix: Use a Set to count distinct book_ids from the returned data:
```typescript
queryFn: async () => {
  const { data, error } = await supabase
    .from("user_progress")
    .select("book_id")
    .eq("user_id", user!.id);
  if (error) throw error;
  const distinctBooks = new Set(data?.map(d => d.book_id));
  return distinctBooks.size;
},
```
Note: The upsert on `user_id,book_id` should already ensure one row per book, but using Set makes this robust against any edge cases.
