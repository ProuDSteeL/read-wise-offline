1. Create migration `supabase/migrations/20260318000001_make_audio_bucket_private.sql`:

```sql
-- Make audio-files bucket private so files require signed URLs
UPDATE storage.buckets
  SET public = false
  WHERE id = 'audio-files';

-- Remove public access policy if it exists
DROP POLICY IF EXISTS "Audio files are publicly accessible" ON storage.objects;

-- Normalize audio_url column: strip full URL prefix, keep only the storage path
-- Current format: https://xxx.supabase.co/storage/v1/object/public/audio-files/path/to/file.mp3
-- Target format: path/to/file.mp3
UPDATE public.summaries
  SET audio_url = regexp_replace(audio_url, '^https?://[^/]+/storage/v1/object/public/audio-files/', '')
  WHERE audio_url IS NOT NULL
    AND audio_url LIKE 'http%';
```

2. Create `supabase/functions/get-audio-url/index.ts`:
   - Follow the existing `send-notifications` pattern exactly: same import style (`https://esm.sh/@supabase/supabase-js@2`), same corsHeaders (copy the full header list), same CORS OPTIONS handling
   - Accept POST with JSON body `{ bookId: string }`
   - Extract user from Authorization header using anon-key client + `auth.getUser()`
   - Create service_role admin client
   - Fetch user profile from `profiles` using adminClient to check subscription
   - Determine if user is Pro: `subscription_type` is `'pro_monthly'` or `'pro_yearly'` AND (`subscription_expires_at` is null OR `subscription_expires_at > new Date().toISOString()`)
   - If NOT Pro: return 403 `{ error: "Pro subscription required for audio" }`
   - If Pro: fetch summary from `summaries` by `book_id` to get the `audio_url` (now a path like `audio/book-id.mp3`)
   - If no audio_url on summary: return 404 `{ error: "No audio available" }`
   - Generate signed URL: `adminClient.storage.from('audio-files').createSignedUrl(audioPath, 86400)` (86400 seconds = 24 hours)
   - Return `{ signedUrl: data.signedUrl, expiresIn: 86400 }`
   - All responses include `corsHeaders` and `Content-Type: application/json`
